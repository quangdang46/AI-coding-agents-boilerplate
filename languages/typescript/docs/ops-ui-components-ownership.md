# TypeScript MCP, permissions, shell, tasks, and teams UI ownership

The archived ops-facing component files in bead `aicd-3ix.4.15.1.3` are not preserved as a one-for-one shipped UI package in the current AICD TypeScript proving slice.

The shipped TypeScript pack keeps long-term ownership for conversation runtime, prompt/config/tool policy, and message orchestration under `languages/typescript/runtime/`, while advanced MCP, team-memory, task workflow, and interactive clarification behavior lives under optional feature-pack families such as `mcp-integration`, `team-memory`, `prompt-suggestion-services`, and `interactive-clarification-tools`. These archived ops-facing components therefore need explicit conceptual-runtime, future-feature-pack, or archive-only disposition instead of being treated as shipped one-for-one UI.

## Shipped-language-pack rule

This subset is complete when each archived ops-facing component row has an explicit shipped-owner or archive-only rationale. These archived UI files must not be mistaken for already-shipped TypeScript runtime modules just because the proving slice preserves their underlying MCP, team, task, shell, and permission responsibilities elsewhere.
