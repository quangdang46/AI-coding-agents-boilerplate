# TypeScript prompt input, messages, and conversation component ownership

The archived prompt-input and message-oriented component files in bead `aicd-3ix.4.15.1.2` are not preserved as a one-for-one shipped UI package in the current AICD TypeScript proving slice.

The shipped TypeScript pack keeps long-term ownership for conversation runtime, session lifecycle, and message orchestration under `languages/typescript/runtime/engine/`, prompt composition and prompt file loading under `languages/typescript/runtime/prompts/`, and optional memory/suggestion behaviors under declared feature packs such as `team-memory` and `prompt-suggestion-services`. These archived component files therefore need explicit conceptual-runtime, future-feature-pack, or archive-only disposition rather than being implied as shipped one-for-one UI.

## Shipped-language-pack rule

This subset is complete when each archived prompt/message/conversation row has an explicit shipped-owner or archive-only rationale. These archived UI files must not be mistaken for already-shipped TypeScript runtime modules just because the proving slice preserves their underlying conversation, prompt, or memory responsibilities elsewhere.
